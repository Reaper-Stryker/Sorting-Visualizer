Bubble Sort is one of the simplest sorting algorithms. It repeatedly compares adjacent elements in a list and swaps them if they are in the wrong order. This process continues until the list is sorted. Start at the beginning of the list. Compare the first element with the second. If the first is greater than the second, swap them. Move to the next pair and repeat the comparison and swapping process. Continue this for all elements in the list. After one pass, the largest element "bubbles up" to its correct position at the end of the list. Repeat the above steps for the remaining unsorted portion of the list until no swaps are needed. Time Complexity : O(N2) Space Complexity : O(1)

Selection Sort is a simple comparison-based algorithm. It divides the input list into two parts: a sorted sublist built up from left to right, and an unsorted sublist occupying the rest of the list.Start with the entire list considered as unsorted.Scan the unsorted portion of the list to find the absolute minimum value.Swap this minimum value with the first element of the unsorted portion.The sorted boundary now moves one element to the right.Repeat the scanning and swapping process for the remaining unsorted portion until the entire list is sorted.Time Complexity: $O(n^2)$Space Complexity: $O(1)$

Insertion Sort builds the final sorted array one item at a time. It behaves similarly to how a person might sort a hand of playing cards.Assume the first element is already sorted.Pick the next element and compare it to the elements in the sorted portion, moving backwards from right to left.Shift all elements in the sorted portion that are strictly greater than the picked element one position to the right.Insert the picked element into its newly vacated, correct position.Repeat this process until all elements have been inserted into their correct sorted places.Time Complexity: $O(n^2)$Space Complexity: $O(1)$

Merge Sort is an efficient, general-purpose sorting algorithm based on the divide-and-conquer strategy.Divide the unsorted list exactly in half to create two sub-lists.Continue dividing the sub-lists recursively until every sub-list contains only a single element (a list of one is considered sorted).Begin merging the sub-lists back together by comparing the first elements of two lists and taking the smaller one.Continue merging and ordering these sub-lists until only one sorted list remains, which is the final output.Time Complexity: $O(n \log n)$Space Complexity: $O(n)$

Quick Sort is a highly efficient divide-and-conquer algorithm that relies on a central pivot point to partition the array.Select an element from the list to serve as the "pivot" (often the last element or a random element).Reorder the list so that all elements with values less than the pivot come before it, and all elements with values greater than the pivot come after it.The pivot is now strictly in its final sorted position.Recursively apply the above steps to the sub-list of elements with smaller values and the sub-list of elements with greater values.Time Complexity: $O(n \log n)$Space Complexity: $O(\log n)$

Heap Sort is a comparison-based sorting technique based on a Binary Heap data structure. It is similar to Selection Sort but uses a heap to find the maximum value faster.Treat the initial list as a binary tree and transform it into a "Max-Heap" structure (where the parent node is always larger than its children).Take the root of the Max-Heap (which is guaranteed to be the largest number) and swap it with the last item in the list.Conceptually remove this largest item from the heap (it is now sorted at the end of the array).Rebuild the Max-Heap structure for the remaining elements to push the next largest number to the root.Repeat the swapping and rebuilding process until the heap size is reduced to zero.Time Complexity: $O(n \log n)$Space Complexity: $O(1)$

Radix Sort avoids direct comparison by sorting numbers digit by digit, from least significant to most significant.Find the largest number in the list to determine the maximum number of digits.Start sorting the numbers based strictly on their least significant digit (the ones place) using a stable subroutine like Counting Sort.Move to the next digit (the tens place) and re-sort the entire list based solely on that digit.Repeat this process for every digit position up to the most significant digit. Once the final digit is processed, the list is completely sorted.Time Complexity: $O(d \cdot (n + k))$Space Complexity: $O(n + k)$

Counting Sort is an integer sorting algorithm that operates without comparing elements. Instead, it relies on counting the occurrences of distinct elements.Find the maximum value in the input list to determine the range of the data.Create an auxiliary "count" array of that size, initially filled with zeros.Iterate through the input list, using the element's value as an index in the count array to increment its tally.Modify the count array by adding the previous counts to the current count, turning the tallies into actual positional indexes.Iterate backwards through the initial list, placing elements into their final sorted array based on the indexes in the count array.Time Complexity: $O(n + k)$Space Complexity: $O(n + k)$

Shell Sort is an optimization of Insertion Sort that allows the exchange of items that are far apart. It uses a sequence of gaps to determine which elements to compare and sort.Start with a large gap and sort the elements at that gap distance using a modified insertion sort.Gradually reduce the gap and repeat the sorting process for each new gap until the gap is reduced to 1, at which point the list is fully sorted.Time Complexity: $O(n^{1.5})$ (depends on the gap sequence)Space Complexity: $O(1)$

Bucket Sort is a distribution sort that scatters elements into several "buckets" based on their value ranges.Create an array of initially empty "buckets" (sub-arrays).Iterate through the original list and distribute each element into a specific bucket based on a mathematical range calculation (e.g., placing values 0-9 in bucket 1, 10-19 in bucket 2).Sort each non-empty bucket individually. This is usually done using Insertion Sort.Visit the buckets in order and concatenate their sorted contents back into the original array.Time Complexity: $O(n + k)$Space Complexity: $O(n + k)$

Linear SearchLinear Search is the most basic searching algorithm. It sequentially checks each element in a list until a match is found or the whole list has been searched.Start at the first element of the list.Compare the current element with the target value. If it matches, return the current index.If it does not match, move to the next element in the list.Continue this process until the target is found or the end of the list is reached.Time Complexity: $O(n)$Space Complexity: $O(1)$

Binary Search is a highly efficient algorithm for finding an item from a sorted list of items. It works by repeatedly dividing in half the portion of the list that could contain the item.Start by identifying the middle element of the sorted list.Compare the target value to the middle element. If they match, return the index.If the target is smaller than the middle element, narrow the search to the left half of the list.If the target is larger, narrow the search to the right half.Repeat this halving process until the target is found or the search space is empty.Time Complexity: $O(\log n)$Space Complexity: $O(1)$

Jump Search is an algorithm for sorted arrays that checks fewer elements than Linear Search by jumping ahead by fixed steps.Calculate a jump step size, typically the square root of the list's length.Jump forward by the step size until you reach an element that is larger than the target value.Once a larger element is found (or the end is reached), perform a standard Linear Search backwards within that previous block.Repeat the linear comparison until the target is found or the block ends.Time Complexity: $O(\sqrt{n})$Space Complexity: $O(1)$

Interpolation Search is an improvement over Binary Search for instances where the values in a sorted array are uniformly distributed. It estimates the position of the target like a human searching a dictionary.Calculate the probable position of the target based on the values at the low and high bounds of the list.Probe that calculated index. If it matches the target, return the index.If the target is smaller than the probed value, calculate a new position in the lower sub-list.If the target is larger, calculate a new position in the upper sub-list.Repeat until the target is found or the calculated bounds cross each other.Time Complexity: $O(\log(\log n))$Space Complexity: $O(1)$

Exponential Search determines a range where the search key may be present, and then performs a Binary Search within that specific range.Check if the first element is the target. If not, set an index pointer to 1.Repeatedly double the index pointer (1, 2, 4, 8...) as long as the element at that index is less than the target.Once the element at the index is greater than the target, you have found a bounded range.Perform a standard Binary Search between the previous index and the current index to find the exact position.Time Complexity: $O(\log n)$Space Complexity: $O(1)$
